第三篇：核心的飞跃——类与对象 (Classes & Objects)

欢迎来到我们系列教程的第三篇！如果你已经掌握了Java的基础语法和流程控制，那么恭喜你，你已经具备了让计算机“做事”的能力。但从现在开始，我们将进行一次思维上的巨大飞跃：从思考“怎么做”，转向思考“让谁来做”。这，就是面向对象编程（Object-Oriented Programming, OOP）的精髓。

1. 面向对象编程入门

在过程式编程中，我们的代码就像一份详细的菜谱，严格按照步骤一步步执行。但在面向对象的世界里，我们不再是厨师，而是变成了餐厅老板。我们会定义不同角色的“员工”（对象），比如厨师、服务员、收银员，并让他们各司其职，协同工作。

类 (Class) 与 对象 (Object)：从蓝图到实例

要理解面向对象，首先必须弄清“类”和“对象”这两个核心概念。

  • 类 (Class)：可以理解为一张 “蓝图” 或一个 “模板”。它定义了一类事物共同拥有的 属性（特征） 和 行为（能做什么）。例如，我们可以设计一张“汽车”的蓝图，规定所有汽车都应该有品牌、颜色这些属性，也应该能启动、加速这些行为。类本身是抽象的，不占用内存。

  • 对象 (Object)：是根据“类”这张蓝图创造出来的 “实体” 或 “实例” (Instance)。每一辆具体的、看得到摸得着的汽车（比如“我那辆红色的法拉利”、“邻居那辆黑色的雅阁”）都是一个对象。对象是具体的，它在内存中真实存在，并拥有自己独特的属性值（我的车是红色，邻居的车是黑色）。

简单总结：类是对象的模板，对象是类的实例。

2. 对象的创建与初始化

在上面的例子中，我们使用 new Car() 来创建对象。这个过程背后，其实是 构造器 (Constructor) 在工作。

构造器 (Constructor)：对象的“出生证明”

构造器是一个特殊的方法，它的作用就是在对象被创建时进行 初始化 工作（比如给属性赋初始值）。它就像是对象的“出生仪式”，在对象诞生的一刻被自动调用，且只调用一次。

构造器有以下特点：

  1. 方法名必须与类名完全相同。
  2. 没有返回值类型，连 void 都不写。

如果我们不写任何构造器，Java会贴心地为我们提供一个看不见的、无参数的默认构造器。但我们通常会自定义构造器，来满足更灵活的初始化需求。

this 关键字：明确指出“我”是谁

当构造器或方法的参数名与类的成员变量名相同时，程序会如何区分呢？这时就需要 this 关键字。this 代表 “当前对象” 的引用。

this 的一个核心用途就是区分同名的成员变量和局部变量。

这种写法是Java开发中的标准规范，非常常用。

3. 封装：面向对象的第一大特性

想象一下，汽车的内部结构非常复杂，如果把所有线路和零件都暴露给驾驶员，驾驶员可能会不小心弄坏它。所以汽车厂商会用一个外壳把内部结构“包”起来，只提供方向盘、油门、刹车等有限的接口给驾驶员。

这个“包装”的思想，在OOP中就叫 封装 (Encapsulation)。

封装的好处：

  • 安全性：保护数据，防止被随意修改。
  • 易用性：隐藏复杂细节，只暴露简单的操作接口。

在Java中，我们主要使用 访问修饰符 来实现封装，最常用的就是 private。

  • private (私有的)：被 private 修饰的成员（属性或方法），只能在本类内部被访问。

现在，外部无法直接操作 age 属性，必须通过我们提供的 setAge 和 getAge 方法，这就保证了数据的安全性和可控性。

4. static 关键字：属于“类”的成员

通常情况下，类的属性和方法是属于 对象 的，每个对象都有一份自己的副本（比如每个Car对象都有自己的color）。但有时，我们希望某个属性或方法是 所有对象共享的，或者说，它不依赖于任何具体对象，而是直接属于 类 本身。这时，就要用到 static 关键字。

被 static 修饰的成员，我们称之为 静态成员 或 类成员。

静态变量 (Static Variables)

所有该类的对象共享同一个静态变量。任何一个对象修改了它的值，其他对象看到的值也会跟着改变。

静态方法 (Static Methods)

静态方法同样不依赖于任何对象，可以直接通过 类名调用。它通常用于实现一些工具类的功能。我们最熟悉的 main 方法就是一个静态方法。

注意： 静态方法内部不能使用 this 关键字，也不能直接访问非静态成员，因为它不与任何具体对象绑定。

一个经典的静态方法应用场景就是 工具类 (Utility Class)。

MathUtil 这样的工具类，它的所有方法都是静态的，我们根本不需要创建它的对象，直接用类名调用即可，非常方便。

到这里，我们已经完成了从过程式到面向对象最关键的思维转变。我们学会了如何用 class 设计蓝图，用 new 创建对象，并通过封装、构造器、this 和 static 等核心知识点，让我们的代码变得更加结构化、安全和易于管理。

掌握好类与对象，就等于拿到了进入面向对象世界的钥匙。在下一篇中，我们将继续探索OOP的另外两大支柱：继承与多态，去构建更加庞大和灵活的程序体系！
